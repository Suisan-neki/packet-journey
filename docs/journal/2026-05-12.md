# 2026-05-12

```bash
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
```

Rust の公式インストーラが入り、`rustup` / `rustc` / `cargo` をバージョン管理しながら使える。

```bash
source "$HOME/.cargo/env"
```

インストール直後のこのターミナルで `cargo` などのパスが通る（新しいウィンドウなら通常は不要）。

```bash
rustup install stable
rustup toolchain install nightly --component rust-src
```

普段用の stable と、eBPF 向けの nightly＋`rust-src` が揃う。あとから回す Aya / bpf 系ビルドの前提。

```bash
cargo install bpf-linker
```

Rust の eBPF をリンクしてバイトコードにまとめる `bpf-linker` が手元にある。

```bash
cargo install cargo-generate
```

GitHub のテンプレから変数を埋めてプロジェクトのひな形を生成できる。

```bash
cargo generate --name xdp-hello -d program_type=xdp https://github.com/aya-rs/aya-template
```

質問の NIC は `eth0` にした。`xdp-hello/` 以下に userspace と eBPF のクレートができた。XDP は Linux 上の話なので、macOS だけだと VM やクロスコンパイルなど別途がいる。

Mac 上では `aya` がそのままビルドできなかった。Docker より先に、**Multipass** というツールで Ubuntu の VM を丸ごと立てるやり方を試すことにした（Canonical が出しているやつで、手元の Mac から簡単に Linux を用意できるらしい）。

```bash
brew install multipass
```

これで Multipass 本体が入る。次のセッションから VM を起動して、その中で Rust ツールチェーンやプロジェクトを用意する。
