## なぜ TCP で中継するのか

最初に詰まったのは「eBPF は VM、GUI は Mac」という物理的な分離。

```text
[Linux VM]  XDP/eBPF はカーネル機能なので VM の中でしか動かない
[macOS]     Tauri の GUI ウィンドウは Mac 側でしか出せない
```

Tauri のイベント機能（`window.emit`）は同じプロセス内の話なので、そのままでは VM 側のデータをフロントに渡せない。そこで間を **TCP + NDJSON（1行1イベントの JSON）** でつないだ。

```text
[VM]  RingBuf → ユーザー空間 → TCP 127.0.0.1:9000 へ NDJSON 配信
        ↓ Lima がゲストのリッスンポートをホストへ自動転送
[Mac] Tauri backend が TCP クライアントで接続 → emit("packet-event") → フロント
```

Lima がリッスンポートをホストへ自動転送してくれるので、Mac 側からは `127.0.0.1:9000` に普通に繋ぐだけでよかった。

## アラートも RingBuf に乗せた

前回の `detect_packet_rate` は閾値超過時に `warn!` ログを出すだけだった。ログは人間には読めるが、GUI に渡すには扱いづらい。

そこで、**フローイベントと同じ RingBuf にアラートも流す**ことにした。`FlowEvent` に `kind`（種別）と `rate`（超過時のパケット数）を足して、1つの構造体で2種類を運ぶ。

```rust
pub const EVENT_KIND_FLOW: u8 = 0;
pub const EVENT_KIND_RATE_ALERT: u8 = 1;

#[repr(C)]
pub struct FlowEvent {
    pub kind: u8,
    pub protocol: u8,
    pub _pad: [u8; 2],
    pub src_addr: [u8; 4],
    pub dst_addr: [u8; 4],
    pub src_port: u16,
    pub dst_port: u16,
    pub rate: u32,
}
```

`#[repr(C)]` でメモリ並びを固定しているので、eBPF 側で書いたバイト列をユーザー空間でそのまま読み戻せる。フィールドを増やしたぶん、ユーザー空間のパース（オフセット）も合わせて直した。

## ユーザー空間は「配信サーバ」になった

`print_flow_event` で黒い画面に出していたデータを、そのまま JSON にして TCP へ投函する形にした。種別は3つ。

```text
flow  : 1パケット検問の記録（proto, src, dst, port）
alert : レート閾値の突破（dst, rate）
stats : 0.5秒ごとの集計（pps, total）
```

PPS（1秒あたり処理パケット数）はユーザー空間側で数えている。`tokio::sync::broadcast` で複数のダッシュボードが同時接続できるようにし、500ms ごとのタイマーで `stats` を流す。

```text
flow を数える → 0.5秒ごとに差分を見る → 2倍して pps に換算 → 配信
```

## フロントは Vanilla + Canvas にした

ReactやVueは便利だが、毎秒大量のパケットが降ってくると仮想DOMの再計算がボトルネックになりやすい。今回は「遅延ゼロで魅せる」が目的なので、**素のJS + Canvas API** にした。

作った要素は3つ。

```text
1. パケットストリーム・タコメーター（Canvas の針付きメーター、pps を可視化）
2. アタック・アラート・フラッシュ（alert 受信で画面全体が赤く点滅 + ログ）
3. 「負荷ゼロ」の CPU 比較バー（従来型 vs eBPF）
```

VM が無くても審査で見せられるように、フロント側に**デモデータ（合成トラフィック）トグル**も付けた。これは乱数で作っているだけで、本物のパケットではない。

## 画面は2種類に分ける方針

フロントは目的別に2つに分けることにした。

```text
画面A: 技術デモ用    → 爆速・派手・リアルタイム（今日作ったのはこっち）
画面B: 医療従事者用  → 視認性・安心感・標準UI（後日、React で追加予定）
```

裏側の eBPF / Rust は 100% 共有のまま、Tauri のマルチウィンドウで HTML/JS を出し分ければよい。なので今日は backend 側のウィンドウ label を `main` に明示しておいた。
