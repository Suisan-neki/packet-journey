# 2026-05-13

Multipass で Ubuntu を立てようとしてかなりハマったのでメモ。

```bash
multipass launch 24.04 --name xdp-dev
```

**出たエラー（何度も同じ）**

```text
qemu-system-aarch64: Property 'host-arm-cpu.sme' not found
```

Multipass は裏で QEMU を使い `-cpu host` でホスト CPU に寄せる。M4 あたりの CPU 拡張（SME）と、同梱 QEMU の認識が噛み合わず落ちる、という筋の既知パターン。[canonical/multipass#3842](https://github.com/canonical/multipass/issues/3842) などに似た報告がまとまっている。自分のミスというより組み合わせの問題。

**ついでにハマったこと**

- 一度 `launch` が失敗しても名前だけ残ることがある → `instance "xdp-dev" already exists`
- `multipass list` で `State: Unknown` のまま → 正常起動していない。`delete` → `purge` で掃除してからやり直す。

macOS / Multipass の更新で直る事例もあるが、すぐには通らなかった。

---

**抜け道として Lima（ドライバ `vz`）**

QEMU を経由せず Apple Virtualization で VM を動かす経路。

```bash
limactl start template:ubuntu-lts
```

Lima v2 では `template://` ではなく `template:` の書き方が推奨、という警告が出た。

ログに `Starting the instance "ubuntu-lts" with internal VM driver "vz"` と出て、`READY` のあと:

```bash
limactl shell ubuntu-lts
```

でゲストに入れた。プロンプトが `lima-ubuntu-lts` で、ホスト側の `xdp-hello` がマウントされたパスにいる状態になった。

**ここまでの整理:** XDP 用に「Linux が欲しい」だけなら Multipass に固執しなくてよく、今回は **Lima + vz** で目的の VM に到達できた。

次はこの VM の中で rustup / nightly+rust-src / bpf-linker を入れて、`xdp-hello/` ワークスペースをビルドするところから。
