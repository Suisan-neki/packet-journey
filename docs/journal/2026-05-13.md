# 2026-05-13

## 今日のゴール（事後に整理）

Mac では無理な **Linux 上の XDP** を、手元の VM で **ビルドして NIC に載せ、パケットで eBPF が動く**ところまで通す。インフラとパイプラインが主役で、プログラムの中身の理解はまだ浅いままでよい、という位置づけ。

---

## Multipass → 諦めた理由

```bash
multipass launch 24.04 --name xdp-dev
```

何度も同じエラー。

```text
qemu-system-aarch64: Property 'host-arm-cpu.sme' not found
```

QEMU が `-cpu host` でホストに寄せる一方、CPU 周りの認識が噛み合わない既知系。[canonical/multipass#3842](https://github.com/canonical/multipass/issues/3842) あたりの話。自分の打ち間違いというより **ツールと Mac の相性**。

`launch` が途中で落ちてもインスタンス名だけ残るので `already exists` になったり、`State: Unknown` のゾンビが残ったりした。`delete` → `purge` で掃うのが筋だった。

## Lima（`vz`）で Linux を取りにいく

Multipass は一旦置いて、**Apple Virtualization（`vz`）** で Ubuntu を立てる。

```bash
limactl start template:ubuntu-lts
```

Lima v2 では `template:` の書き方が推奨、という警告が出た。

```bash
limactl shell ubuntu-lts
```

`READY` のあと、プロンプトが `lima-ubuntu-lts` になればゲストに入れている。ここでは **`limactl` は存在しない**（Mac 用）。VM の中で `limactl` を打つと `command not found` になるのは正常。

## VM の中で Rust とビルド環境

Mac に入れた rustup や bpf-linker はそのまま使えないので、ゲストにもう一度入れる。

```bash
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
```

（一度 `| sh` を `| s` にして失敗した。パイプ先の打ち間違い。）

```bash
source "$HOME/.cargo/env"
rustup install stable
rustup default stable
rustup toolchain install nightly --component rust-src
cargo install bpf-linker
```

あわせて `build-essential` や `clang` など、コンパイル用の apt も入れた。

## 共有ディレクトリは書けない

`/Users/.../xdp-hello` が VM から見えても **読み取り専用**で、`target` にロックを置けず `Read-only file system` になった。

**対処:** ホーム以下にコピーしてそこでビルドする。

```bash
cp -a /Users/suisan_2003/xdp-hello ~/xdp-hello
cd ~/xdp-hello/xdp-hello
cargo build --release
```

Mac 側に `scripts/lima-sync.sh` を置いた。**同期と VM 内ビルドは Mac で実行**し、`limactl` と `tar | limactl shell` で `~/xdp-hello` を更新する。VM の中でこのスクリプトを叩くと `limactl が PATH にありません` になるので、**プロンプトが `lima-ubuntu-lts` のときは打たない**。

```bash
cd /Users/suisan_2003/xdp-hello
./scripts/lima-sync.sh --build
```

## XDP のアタッチでつまずいたところ

`unknown network interface eth0`。VM には `eth0` があったが、テンプレの **`default_value = "eth0 "` の末尾スペース**で、デフォルトの IF 名が実質ずれていた。リポジトリ側は `"eth0"` に直した。

---

## 実行まで

VM 内、コピー先のワークスペースで。

```bash
cd ~/xdp-hello/xdp-hello
RUST_LOG=info cargo run --release --config 'target."cfg(all())".runner="sudo -E"'
```

`[INFO xdp_hello] received a packet` が続く。NIC に届くパケットのたびに eBPF が動いているログ。Ctrl+C で止める。

## 今日の一文まとめ

**Mac は司令塔。Linux VM（Lima + vz）の上だけで、Aya の XDP テンプレをビルドして NIC に載せ、パケットで反応するところまで通した。** Multipass の SME、共有 FS の RO、`limactl` はホスト専用、IF 名のスペース、など周辺に時間を使ったが、ゴールの線としてはここまでで一区切り。
